Learning Outcomes

  1. Define spec-driven development and explain its role in a software development workflow.
  2. Describe the "syntax woods" problem and how jumping to code without a plan causes it.
  3. Distinguish between using AI as an architect (a problem decomposer) and using it as a magic code button (a solution generator).
  4. Explain how a specification functions as a step-by-step logical blueprint that guides implementation.
  5. Identify the four components of a technical specification: inputs, outputs, logic, and edge cases.
  6. Write a use case that describes what a user does — step by step — in a proposed system. (OJL 1.b)
  7. Apply requirements language that is precise, testable, and implementer-neutral. (OJL 1.a, 1.c)
  8. Explain how each component of a specification maps to a corresponding implementation step.
  9. Apply prompting strategies that direct AI to decompose a problem into requirements rather than generate a solution. (OJL 1.a)
  10. Evaluate AI-generated specification output by identifying what is precise, what is vague, and what is missing.
  11. Identify edge cases in an AI-generated spec by systematically applying the "what happens when…?" question.
  12. Refine a first-draft specification through at least one full cycle of the prompt–review–refine loop. (OJL 1.c)
  13. Translate each requirement in a specification into a corresponding, ordered implementation step.
  14. Produce an implementation plan — a numbered list of pseudocode or plain-language steps — derived directly from a complete specification. (OJL 1.c)
  15. Apply use cases as a verification checklist to confirm that an implementation plan is complete. (OJL 1.b)
  16. Explain when and how to override or correct AI suggestions in order to remain the decision-maker in the development process.
