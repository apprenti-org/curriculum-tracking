Syllabus: Software Development Lifecycle

Software Development Fundamentals 24 hours DRAFT

Course Description

Software Development Lifecycle introduces learners to the phases and methodologies of professional software development. Students will explore SDLC phases including planning, requirements, design, development, testing, deployment, and maintenance using a Personal Budgeting App as the project thread. The course covers popular SDLC models (Waterfall, Agile, Spiral, V-Model) and introduces Scrum methodology. This course prepares developers to work within structured software processes and understand the relationship between business needs and technical implementation.

Summative Assessment: SDLC Application Project

Students will apply SDLC phases to develop a Personal Budgeting App or similar project. The assessment evaluates understanding of each phase, proper documentation, appropriate methodology selection, and ability to execute each SDLC stage. Learners must demonstrate understanding of how phases interconnect and how different methodologies approach software development differently.

Learning Outcomes

  1. Understand the seven SDLC phases and their purposes
  2. Apply planning and requirements gathering for software projects
  3. Create system designs including architecture and database models
  4. Implement features according to specifications
  5. Plan and execute testing strategies
  6. Manage staging and production deployments
  7. Understand maintenance and support requirements
  8. Compare Waterfall, Agile, Spiral, V-Model, and Iterative approaches
  9. Apply Scrum methodology effectively
  10. Work within structured software development processes
  11. Understand project management in software development

Course Outline

1 Introduction to SDLC 2 hours
Lesson: 1: Software Development Lifecycle Overview
  • Phases of SDLC: Planning, Requirements, Design, Development, Testing, Deployment, Maintenance
  • Structured approach to software development
  • Alignment with business goals
  • Iterative and sequential models
2 SDLC Phases in Practice 16 hours
Lesson: 2: SDLC — Planning (2.5 hours)
  • Define project scope, objectives, and feasibility
  • Identify resources, risks, and budget constraints
  • Create a project roadmap
  • Stakeholder analysis and communication
Lesson: 3: SDLC — Requirements (2.5 hours)
  • Gather functional and non-functional requirements
  • Requirements documentation
  • User stories and acceptance criteria
  • Prioritization and change management
Lesson: 4: SDLC — Design (3 hours)
  • UI/UX design and wireframes
  • System architecture (frontend, backend, database)
  • Database design and data modeling
  • Technical specifications
Lesson: 5: SDLC — Development (3 hours)
  • Writing code according to design specifications
  • Coding standards and best practices
  • Version control and continuous integration
  • Code reviews and collaboration
Lesson: 6: SDLC — Testing (3 hours)
  • Unit testing, integration testing, system testing
  • User Acceptance Testing (UAT)
  • Test-driven development concepts
  • Bug tracking and resolution
Lesson: 7: SDLC — Deployment (2.5 hours)
  • Staging vs production environments
  • Manual vs automated deployment
  • Release notes and documentation
  • Post-deployment monitoring
Lesson: 8: SDLC — Maintenance and Support (2 hours)
  • Corrective, adaptive, perfective, and preventive maintenance
  • Managing updates and bug fixes
  • Performance improvements and user feedback
  • End-of-life planning
3 Methodologies & Frameworks 6 hours
Lesson: 9: Popular SDLC Models (3 hours)
  • Waterfall: sequential, predictable, phase-gated
  • Agile: iterative, flexible, customer-focused
  • Spiral: risk-driven, combines iteration with planning
  • V-Model: testing at each phase
  • Iterative: incremental releases
  • Comparing approaches and trade-offs
Lesson: 10: How to Scrum (3 hours)
  • Scrum fundamentals and Agile values
  • Scrum roles: Product Owner, Scrum Master, Development Team
  • Scrum events: Sprint Planning, Daily Standup, Sprint Review, Sprint Retrospective
  • Scrum artifacts: Product Backlog, Sprint Backlog, Increment
  • Implementing Scrum in real-world projects

Labs & Practical Exercises

  • Project Planning: Define scope, objectives, and requirements for Personal Budgeting App
  • Requirements Documentation: Write user stories and acceptance criteria
  • System Design: Create architecture diagrams and database designs
  • Development: Implement features in iterations
  • Testing Strategy: Plan and execute tests at different levels
  • Deployment Planning: Plan staging and production release
  • Agile Sprint: Execute a two-week sprint using Scrum
  • Methodology Comparison: Choose appropriate methodology for scenarios
  • Project Retrospective: Reflect on lessons learned
