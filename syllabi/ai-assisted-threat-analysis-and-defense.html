Syllabus: AI-Assisted Threat Analysis and Defense

Client Technical Support Engineer (CTSE) 4 hours IN PROGRESS

Course Description

AI-Assisted Threat Analysis and Defense prepares client technical support engineers to leverage generative AI tools in security operations contexts. The course covers extracting and synthesizing threat intelligence from large, unstructured data sets; using AI tools (Copilot, ChatGPT) for code generation, test creation, and secure coding practices including encryption and IAM protocols; and evaluating AI-enhanced operational tools such as IDS/IPS systems and SIEM platforms for accuracy and effectiveness.

The course also addresses the ethical implications of AI-powered data analysis, the importance of rigorous access controls, and the application of responsible AI frameworks for compliance checks. Learners apply these skills to AI-driven incident response documentation and explore automated execution through SOAR playbooks.

This is a supplemental course within the CTSE curriculum (Group 4: AI and Advanced Supports), aligned to OJL competencies 4.e (diagnostic and monitoring tools), 4.f (system logs and traces), and 5.a (compliance concerns).

Learning Outcomes

  1. Use generative AI tools to extract, filter, and synthesize threat intelligence from unstructured data sources such as threat reports and security advisories.
  2. Apply AI-assisted code generation techniques to produce security-relevant code, including implementations that follow encryption and IAM best practices.
  3. Evaluate AI-generated code and security tool output for accuracy, completeness, and potential vulnerabilities.
  4. Assess AI-enhanced IDS/IPS and SIEM tools by interpreting their outputs, identifying false positives, and validating their effectiveness against known threat scenarios.
  5. Apply a responsible AI framework to verify compliance with organizational and regulatory requirements when using AI in security operations.
  6. Use AI to draft incident response documentation and design a basic SOAR playbook for automating a defined security response procedure.

Course Outline

1 AI-Assisted Threat Analysis and Defense 4 hours
Extracting and Synthesizing Threat Intelligence with AI (1h)
  • Threat intelligence landscape: reports, advisories, and unstructured data sources
  • Prompting AI to extract, summarize, and cross-reference threat indicators
  • Evaluating AI output quality and hallucination risks in security contexts
  • Synthesizing threat summaries into actionable briefs
AI-Assisted Secure Coding and IAM Practices (1h)
  • Using AI tools to generate encryption implementations and IAM policy code
  • Reviewing AI-generated code for security vulnerabilities
  • Test creation for security-critical code with AI assistance
  • Ethical considerations and data exposure risks in AI-assisted development
Evaluating AI-Enhanced Security Tools: IDS/IPS and SIEM (1h)
  • How AI is integrated into IDS/IPS and SIEM platforms
  • Interpreting AI-generated alerts: distinguishing true positives from false positives
  • Evaluating tool effectiveness and accuracy against known threat scenarios
  • Access controls and data governance for AI-powered security tooling
Responsible AI for Compliance and Incident Response Automation (1h)
  • Responsible AI frameworks applied to security operations
  • Compliance checks using AI: mapping outputs to regulatory requirements
  • AI-assisted incident response documentation
  • Introduction to SOAR playbooks and AI-driven automation of security responses

Assessment

Each lesson includes a knowledge check (embedded in the SCORM interactive) and a graded quiz. Assessments verify ability to apply AI tools to security operations tasks and evaluate AI output quality.

  • Per-lesson knowledge check (SCORM interactive)
  • Per-lesson graded quiz

Software and Tools

  • Access to a generative AI tool (Copilot or ChatGPT)
  • SIEM/IDS demo environment or instructor-provided simulation scenarios
  • Code editor (VS Code or equivalent) for secure coding exercises
