Syllabus: Python Coding Booster Intensive

Cybersecurity, Operations & Infrastructure, Software Development (optional) 16 hours DRAFT

Course Description

Python Coding Booster Intensive is a two-day intensive program teaching advanced Python development practices in a realistic engineering environment. Students will apply test-driven development (TDD), AI-assisted coding with disciplined workflows, safe modification of existing systems, CI/CD awareness, and DevOps principles. The course uses Scrum simulations and real-world scenarios to teach how to extend Python systems safely, validate AI-generated code, deploy with confidence, and understand security implications. This course prepares engineers to work effectively in professional development environments.

Summative Assessment: Code Quality and Engineering Review

Students will complete two major scenarios: greenfield development and legacy code modification. The assessment evaluates code quality, test coverage, AI prompt quality and validation, safe deployment practices, communication clarity, and system safety awareness. Learners must present their architectural decisions, explain their AI usage strategy, demonstrate TDD discipline, and show how they prevented regressions.

Learning Outcomes

  1. Apply TDD across both greenfield and legacy code
  2. Use AI tools safely with disciplined validation workflows
  3. Understand when and when not to use AI for coding
  4. Identify risks in existing systems before modification
  5. Implement defensive programming and exception handling
  6. Design secure and maintainable Python code
  7. Use Git and GitHub for safe deployment
  8. Understand CI/CD pipelines and their purpose
  9. Recognize DevOps and security considerations in scripting
  10. Communicate architectural decisions effectively
  11. Validate AI-generated code thoroughly
  12. Prevent regressions in production systems

Labs & Practical Exercises

  • Greenfield Project Development: Apply TDD from requirements to deployment
  • AI Workflow Validation: Prompt AI, review output, refactor code
  • Legacy Code Analysis: Understand and modify existing systems
  • Test-Driven Development: Write tests before implementation
  • Defensive Programming: Add error handling and validation
  • Git Workflow: Make commits, create PRs, merge safely
  • CI/CD Understanding: Review pipeline, understand deployment
  • Scrum Simulation: Work in teams with daily standups and reviews
