Syllabus: Non-Relational Data

Data Analyst (DANL) 50 hours DRAFT

Course Description

Non-Relational Data explores NoSQL databases and alternative data storage paradigms. Students will gain practical experience with document databases (MongoDB, Cosmos DB), key-value stores (Azure Table Storage), column stores, and graph databases. The course covers installation of development tools including Python, Jupyter Notebooks, and Docker, and provides hands-on exercises querying different NoSQL platforms. This course prepares learners to select and work with appropriate database technologies for different data scenarios, complementing relational database knowledge.

Summative Assessment: NoSQL Database Implementation Project

Students will implement a complete non-relational data solution using multiple database types. The assessment requires designing and querying document databases, implementing key-value stores, working with columnar databases, and exploring graph relationships. Learners must demonstrate understanding of when to use NoSQL versus relational databases and proficiency querying different platforms using Azure and Python.

Learning Outcomes

  1. Understand NoSQL database concepts and common platforms
  2. Use document databases to store and query structured and semi-structured data
  3. Implement key-value store solutions for fast data retrieval
  4. Identify when to use columnar databases for analytical queries
  5. Design and query graph databases to explore relationships
  6. Use Python with various NoSQL platforms
  7. Implement Azure database solutions (Cosmos DB, Table Storage)
  8. Design appropriate schema for NoSQL databases
  9. Query NoSQL databases using platform-specific tools
  10. Choose appropriate database technologies for different use cases
