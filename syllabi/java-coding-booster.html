Syllabus: Java Coding Booster Intensive

Software Developer 16 hours DRAFT

Course Description

The Java Coding Booster Intensive is a two-day intensive training designed to prepare early-career software developers for real-world engineering workflows in enterprise environments. Learners participate in an authentic Scrum simulation with feature-based development, Test-Driven Development (TDD), and continuous integration via GitHub. Day 1 covers greenfield development and TDD; Day 2 covers legacy systems and hybrid development. AI-assisted coding is mandatory (85% of workflow), shifting focus from syntax mastery to execution, decision-making, and engineering judgment.

Summative Assessment: Production Release & Retrospective

Learners submit a final Pull Request integrating both new services and legacy system modifications. The instructor runs a load test simulation ("Chaos Script") to validate production readiness. Teams participate in a retrospective discussing how to balance AI dependency with practical engineering skills. Assessment evaluates code quality, test coverage (minimum 80%), integration risk assessment, and ability to work within Scrum teams.

Learning Outcomes

  1. Demonstrate consistent Java coding proficiency required for productive early-career developers
  2. Operate effectively within Scrum teams, participating in stand-ups, reviews, and iterative delivery
  3. Apply Test-Driven Development (TDD) practices for reliability and maintainability
  4. Leverage Generative AI tools (GitHub Copilot, ChatGPT) responsibly and effectively
  5. Execute work across greenfield, bug-fix, and hybrid development scenarios
  6. Shift focus from syntax to execution and decision-making using AI tools
  7. Reason about system architecture and integration risk in distributed systems

Labs & Practical Exercises

  • Scrum simulation: greenfield development with feature requirements
  • Create Transaction Validation Service using Spring Boot
  • Implement dependency injection and REST controllers
  • Write JUnit tests and use Mockito for mocking
  • Apply TDD methodology with Red–Green–Refactor cycle
  • GitHub commit and pull request workflow
  • Integration with legacy payment processor system
  • Debugging with AI assistance and stack trace analysis
  • Code refactoring using AI while maintaining test coverage
  • Load testing and production deployment simulation
  • Team standup and retrospective participation
