Course Description

A condensed, 6-hour overview of the source-control basics required by the A-31 Cyber Developer Associate standard. The course teaches what Git and GitHub are and how distributed version control supports collaborative software development, then walks the everyday workflow — initializing a repository, checking changes in and out, pushing to and pulling from upstream remotes, cloning, branching, and collaborating through pull requests. It closes with merge-conflict resolution and credentials hygiene so secrets stay out of version control. The hands-on backbone is a per-lesson instructor-demo code-along; assessment is carried by per-lesson quizzes plus a final cumulative quiz.

## Course-Level Learning Outcomes

1. Explain what Git and GitHub are and how distributed version control supports collaborative software development.

2. Perform the core Git workflow from the command line and GitHub — initialize a repository, stage and commit changes, and check in / check out, push and pull to and from upstream remotes.

3. Clone repositories and work across local and remote repositories.

4. Use branches and pull requests to collaborate on a shared repository.

5. Identify and resolve merge conflicts using both GitHub and the command line.

6. Keep secrets out of version control — apply credentials hygiene with `.gitignore`, safe token handling, and remediation of committed secrets.

Course Outline

1 Git and GitHub Basics
  • Git Background — version control and why it matters; centralized vs. distributed; Git vs. GitHub; core vocabulary; installing and configuring Git.
  • Code-along: install and configure Git, create a first repository (instructor demo).
  • GitHub for Individuals — initialize a repository; stage-and-commit (check in); push/pull with upstream; clone; check out and inspect history; repository structure and README.
  • Code-along: local and remote repositories end-to-end — init → stage → commit → push → pull → clone (instructor demo).
2 Collaboration and Secure Practices
  • GitHub for Groups — branching strategies; creating and connecting branches; pull requests and code review; the end-to-end collaboration workflow.
  • Code-along: create a branch, open a pull request, review and merge (instructor demo).
  • Resolving Merge Conflicts — what causes a conflict; reading diffs and conflict markers; resolving on the command line and on GitHub.
  • Code-along: create and resolve a merge conflict on the command line and on GitHub (instructor demo).
  • Secrets and Credentials Hygiene in Git — why secrets must never be committed; `.gitignore` for credentials and environment files; safe token handling and remediating a committed secret.
  • Code-along: add a `.gitignore` to keep a secret out of a repo, and remove a credential that was already committed (instructor demo).
  • Formative — quizzes: one per lesson, `target_questions = ceil(5 × lesson hours)`.
  • Formative — code-alongs: one instructor-demo code-along per lesson (demonstrated and followed, not graded).
  • Summative: a final cumulative quiz covering all five lessons.
