Syllabus: HTTP Services

Software Development Fundamentals DRAFT

Course Description

HTTP Services introduces learners to the foundational principles of internet communication and web services. Students will understand the difference between the internet and the web, learn how HTTP requests and responses work, explore JSON as a data interchange format, and master RESTful API design principles. This course provides essential knowledge for building web applications, consuming APIs, and understanding modern service-oriented architecture. The content applies across all development tracks and serves as a prerequisite for web development courses.

Summative Assessment: HTTP Protocol and REST API Analysis

Students will demonstrate understanding of HTTP fundamentals through exercises analyzing HTTP requests and responses, mapping CRUD operations to HTTP methods, and interpreting API documentation. The assessment requires ability to sketch valid HTTP requests and responses with JSON payloads, understand protocol layers and network stacks, and explain REST design principles. Learners must apply this knowledge to real-world API scenarios.

Learning Outcomes

  1. Differentiate between the internet and the web
  2. Describe the TCP/IP protocol stack and how protocols layer
  3. Diagram a minimum web application architecture
  4. Explain the purpose and benefits of web services
  5. Describe HTTP request and response structure
  6. Map CRUD operations to HTTP methods (GET, POST, PUT, DELETE)
  7. Categorize HTTP status codes and explain their meaning
  8. Explain the purpose of HTTP headers
  9. Understand JSON serialization and object notation
  10. Apply RESTful design principles to API design
  11. Consume and test APIs using developer tools
