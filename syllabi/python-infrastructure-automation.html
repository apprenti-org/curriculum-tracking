Syllabus: Python for Infrastructure Automation

Operations Support Specialist — Network (Course 7, RTI Area 3) 32 hours CURRENT

Course Description

Python for Infrastructure Automation teaches IT operations professionals how to use Python to automate routine operational tasks, manage cloud infrastructure, and improve system reliability. This course builds on Python fundamentals and applies them to real-world infrastructure challenges including configuration management, monitoring, backup automation, and deployment processes. Learners will develop practical skills in working with APIs, cloud provider SDKs, and automation frameworks essential for modern operations teams.

Learning Outcomes

  1. Use `pathlib` to traverse and manipulate filesystem paths.
  2. Read and write files with appropriate modes, encoding, and context managers.
  3. Read environment variables and apply defaults when variables are missing.
  4. Execute subprocesses safely, capturing output and exit codes.
  5. Apply `re` module functions (`match`, `search`, `findall`, `finditer`, `sub`) to common text tasks.
  6. Construct regex patterns using anchors, groups, and quantifiers.
  7. Use named groups and `finditer` to parse structured data from logs.
  8. Analyze a real log file end-to-end and extract operational metrics.
  9. Describe the socket abstraction at a high level (TCP vs UDP, address families).
  10. Use the `requests` library to call HTTP and HTTPS APIs.
  11. Handle common request patterns: headers, JSON, authentication, timeouts.
  12. Build a basic network-diagnostic script that checks a list of endpoints.
  13. Parse configuration files in JSON, YAML, and INI formats.
  14. Validate configuration against a schema using `jsonschema`.
  15. Apply configuration across multiple hosts idempotently.
  16. Handle environment-specific config overrides through layering.
  17. Collect system metrics using `psutil` and OS tools.
  18. Design health checks with timeouts, retries, and clear failure criteria.
  19. Dispatch alerts via webhooks, SMTP, and Slack.
  20. Integrate monitoring into an ops workflow with thresholds and noise control.
  21. Design backup scripts for files, databases, and cloud resources.
  22. Schedule backup jobs with cron expressions or systemd timers.
  23. Verify backups through automated restore testing.
  24. Document recovery procedures in a runbook callable from Python.
  25. Automate provisioning of servers and services.
  26. Deploy applications with managed dependencies and repeatable steps.
  27. Implement rolling-update and zero-downtime deployment patterns.
  28. Roll back a failed deployment safely.
  29. Interact with AWS services programmatically using `boto3`.
  30. Compare Azure SDK and GCP SDK patterns to `boto3`.
  31. Manage credentials securely (roles, env vars, profiles).
  32. Handle pagination, retries, and rate limits in SDK calls.
  33. Describe IaC concepts and compare to imperative scripting.
  34. Use Pulumi to define AWS resources in Python.
  35. Manage Pulumi stacks and state backends.
  36. Execute the preview → up → destroy workflow.
  37. Send custom metrics to CloudWatch and Azure Monitor.
  38. Aggregate logs from multiple sources into a centralized pipeline.
  39. Structure logs for automated analysis (JSON, levels, correlation IDs).
  40. Create dashboards and alarms programmatically.
  41. Write tests for automation scripts with `pytest`.
  42. Structure a Python project for reproducible CI runs.
  43. Integrate scripts into a GitHub Actions CI pipeline.
  44. Gate deployments on test outcomes with environment approvals.
  45. Apply retry patterns with exponential backoff and jitter.
  46. Implement the circuit-breaker pattern for external dependencies.
  47. Design graceful-degradation fallbacks.
  48. Log errors with enough context for downstream investigation.
  49. Distinguish threading, multiprocessing, and asyncio patterns in Python.
  50. Apply threading (via `concurrent.futures`) for I/O-bound workloads.
  51. Apply multiprocessing for CPU-bound workloads.
  52. Use `asyncio` and `aiohttp` for high-concurrency network operations.
  53. Retrieve secrets from AWS Secrets Manager and HashiCorp Vault.
  54. Encrypt data at rest using the `cryptography` library.
  55. Handle credentials securely in scripts and avoid common pitfalls.
  56. Audit automation scripts against a security checklist.
  57. Emit structured logs with consistent schema and correlation IDs.
  58. Add distributed tracing with OpenTelemetry to automation scripts.
  59. Document scripts for ops consumers (runbooks, help flags, exit codes).
  60. Measure and profile automation performance.

Course Outline

1 Python for Systems Administration 6 hours
Lesson: 1: File and System Operations (2h)
  • File I/O: text, binary, and context managers
  • Directory and path operations with `pathlib`
  • Reading environment variables with `os.environ`
  • `subprocess` for shelling out safely (OJL 3a)
  • Pair coding: directory-audit script using `pathlib` + `subprocess`
Lesson: 2: Regular Expressions and Text Processing (2h)
  • `re` module basics: `match`, `search`, `findall`, `finditer`, `sub`
  • Pattern construction: anchors, groups, quantifiers
  • Named groups and `finditer` for parsing structured log data
  • Log-file analysis end-to-end
  • Individual: write a log analyzer extracting per-status-code counts and top client IPs
Lesson: 3: Network Programming Basics (2h)
  • Sockets primer: TCP vs UDP, address families
  • HTTP with the `requests` library
  • Headers, JSON, authentication, timeouts
  • Network-diagnostic scripts
  • Individual: URL health checker handling timeouts and connection errors
2 Infrastructure Automation Scripts 8 hours
Lesson: 4: Configuration Management Scripts (2h)
  • Parsing JSON, YAML, INI
  • Config validation with `jsonschema`
  • Bulk apply across hosts idempotently (OJL 3e)
  • Environment-specific overrides through layering
  • Pair coding: inventory-apply script with `--dry-run` default
Lesson: 5: Monitoring and Alerting Scripts (2h)
  • System metrics with `psutil`
  • Health-check patterns: timeouts, retries, failure criteria
  • Alert dispatching: webhooks, SMTP, Slack
  • Monitoring workflow integration with thresholds and noise control
  • Individual: disk-usage monitor with webhook alert and alert deduplication
Lesson: 6: Backup and Disaster Recovery Automation (2h)
  • Backup patterns: files, databases, cloud resources (OJL 3a)
  • Scheduling with cron and systemd timers
  • Restore-test automation (OJL 4e)
  • DR runbooks in Python
  • Individual: backup-and-verify script with SHA256 checksum validation
Lesson: 7: Deployment and Provisioning Scripts (2h)
  • Provisioning automation (OJL 3e)
  • Application deployment scripts
  • Rolling-update and zero-downtime patterns
  • Rollback strategies
  • Pair coding: deploy + rollback script using a symlinked "current" release directory
3 Working with APIs and Cloud Platforms 8 hours
Lesson: 8: RESTful APIs and Cloud SDKs (2h)
  • `boto3` fundamentals for AWS
  • Azure SDK and GCP SDK parity examples
  • Credentials: env vars, IAM roles, profiles
  • Pagination, retries, rate limits
  • Individual: S3 inventory report using `boto3` paginators
Lesson: 9: Infrastructure as Code with Python (2h)
  • IaC concepts: Pulumi vs Terraform vs CDK
  • Pulumi Python resources: S3, IAM, EC2
  • Stacks and state management
  • Preview → up → destroy workflow
  • Individual: mini Pulumi stack with S3 bucket and restricted IAM policy
Lesson: 10: Monitoring and Logging Integration (2h)
  • Custom metrics: CloudWatch and Azure Monitor
  • Log aggregation patterns
  • Structured logging: JSON, levels, correlation IDs
  • Dashboards and alarms programmatically
  • Individual: emit a custom CloudWatch metric and attach an alarm
Lesson: 11: Continuous Integration with Python (2h)
  • `pytest` basics for automation scripts
  • Project structure: packages, dependencies, lock files
  • GitHub Actions pipeline
  • Deploy gates and environment approvals
  • Pair coding: wire `pytest` into a GitHub Actions workflow using mocks for external calls
4 Advanced Automation and Best Practices 8 hours
Lesson: 12: Error Handling and Resilience (2h)
  • Retry with exponential backoff using `tenacity`
  • Circuit-breaker pattern
  • Graceful degradation strategies
  • Error logging with diagnostic context
  • Individual: wrap an API call with `tenacity` retries and a circuit breaker
Lesson: 13: Concurrency and Parallel Processing (2h)
  • When to use what: GIL, I/O vs CPU-bound workloads
  • Threading and `concurrent.futures`
  • Multiprocessing
  • Asyncio and `aiohttp`
  • Individual: fetch 100 URLs three ways (sequential, threaded, asyncio) and compare
Lesson: 14: Security and Secret Management (2h)
  • Secrets manager patterns: AWS Secrets Manager, HashiCorp Vault
  • Encryption at rest with the `cryptography` library
  • Credential hygiene in scripts
  • Security audit checklist
  • Individual: secret-backed API call verifying the key never leaks to logs
Lesson: 15: Production Automation Patterns (2h)
  • Structured logging patterns
  • Distributed tracing with OpenTelemetry
  • Documentation conventions: runbooks, help flags, exit codes
  • Profiling automation performance
  • Individual: instrument an existing script with logs, spans, and a profiler run
5 Capstone Project 2 hours
